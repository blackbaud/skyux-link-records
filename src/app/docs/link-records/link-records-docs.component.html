This link records component shows two records and provides options for suggesting a match or adding a new record.

To minimize cognitive load, comparison cards should display the minimum data required to uniquely identify a record and allow users to create a record when no true match is found.

The intent of this component is to allow users to compare records when a single record is very likely the correct match. To present users with a set of less-certain matches, use a different approach.

Record linking is frequently handled in the context of importing multiple records. In these cases, you can repeat the component in a vertical list of comparisons, as in the demo above. If you import multiple types of records, divide the task by record type so that users work through multiple lists that each include a single record type.
